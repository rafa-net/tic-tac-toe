This is my project. I am going to exhaust all the possibilities of winning and losing at this game, so I can set it up in programming correctly.

  • From left to right, enumeration begins with column one, two and three.
  • From top to bottom, enumeration begins with row one, two and three.
  • This is counterintuitive to the source HTML, but consistent with the DOM.
  1. When the game begins, the first player is 'X'. The objective is to make three consecutive marks in manner such:
    • Horizontally
    • Vertically
    • Diagonally
  • Each pink square is a cell, and the first one is numbered zero (0).
  • This is because of zero-based indexing, which means that computers start counting from zero (0).
  • So we introduce the concept of array, which enumerates the cells like so: cells[0], cells[1], cells[2], cells[3], cells[4], cells[5], cells[6], cells[7], cells[8].
  • This enumeration, because of the DOM, starts from top to bottom, and goes left to right.
